# Project Rules

## `book/` directory policy

The `book/` directory is exclusively for publishable manuscripts. Do not create working files there.

Working files (editorial summaries, review notes, scratch drafts, source verification checklists) must go in `drafts/` at the repo root.

## Git commit rules

- Never include `Co-Authored-By` trailers in commits
- Never attribute commits to Claude or any AI in any way
- Commits must appear solely authored by the user
